Security Policy

Responsible Disclosure

I take security seriously and appreciate the security research community's efforts to help keep my website secure.

Reporting Security Issues

If you discover a security vulnerability, please:

  • Email me directly at vipul@vipulworks.com
  • Provide a detailed description of the issue
  • Include steps to reproduce the vulnerability
  • Allow reasonable time for response and fix
Contact: vipul@vipulworks.com
Response Time: Within 48 hours
Scope: vipulworks.com and related subdomains

What I Consider Security Issues

  • Cross-site scripting (XSS)
  • Cross-site request forgery (CSRF)
  • SQL injection
  • Authentication bypass
  • Data exposure
  • Server-side code execution

What I Don't Consider Security Issues

  • Missing security headers (unless exploitable)
  • Banner disclosure
  • Open redirects (unless exploitable)
  • Rate limiting
  • Missing CSP headers

Recognition

Security researchers who responsibly disclose vulnerabilities will be acknowledged on this page (with permission).
